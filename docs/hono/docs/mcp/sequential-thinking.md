# Sequential Thinking

> 복잡한 문제를 단계별로 분석하는 사고 도구

---

## 개요

Sequential Thinking은 복잡한 문제를 체계적으로 분석하고 해결하기 위한 MCP 도구입니다.

---

## 사용 시점

### ✅ 사용 권장
- **복잡한 버그 디버깅**: 원인 파악이 어려운 버그
- **아키텍처 분석/설계**: 시스템 구조 이해 및 설계
- **성능 병목 분석**: 성능 문제 원인 파악
- **리팩토링 계획**: 대규모 코드 변경 계획
- **보안 취약점 분석**: 보안 이슈 탐지 및 해결

### ❌ 사용 불필요
- 단순한 코드 작성
- 명확한 버그 수정
- 간단한 질문 응답

---

## 분석 프로세스

```
1. 문제 정의
   └─ 무엇을 해결해야 하는가?

2. 정보 수집
   └─ 관련 코드, 로그, 에러 메시지

3. 가설 설정
   └─ 가능한 원인들 나열

4. 가설 검증
   └─ 각 가설을 코드/데이터로 검증

5. 결론 도출
   └─ 검증된 원인과 해결책

6. 실행 계획
   └─ 구체적인 수정 단계
```

---

## 예시 상황

### 복잡한 버그 디버깅
```
문제: "API 응답이 간헐적으로 느림"

Sequential Thinking 활용:
1. 느린 요청의 패턴 분석
2. 가능한 원인 나열 (DB, 네트워크, 코드)
3. 각 원인별 증거 수집
4. 병목 지점 확인
5. 해결책 제시
```

### 아키텍처 분석
```
문제: "마이크로서비스 분리 계획"

Sequential Thinking 활용:
1. 현재 모놀리스 구조 분석
2. 도메인 경계 식별
3. 의존성 그래프 작성
4. 분리 우선순위 결정
5. 마이그레이션 계획 수립
```

---

## 장점

### 1. 체계적 접근
- 단계별 사고 과정 기록
- 놓치는 부분 최소화

### 2. 가설 기반 분석
- 추측이 아닌 검증 기반
- 명확한 근거 제시

### 3. 재사용 가능
- 분석 과정 문서화
- 유사 문제에 적용 가능
