# UI/UX 디자인 가이드

> **상위 문서**: [홈](../index.md)

TanStack Start + Tailwind CSS 프로젝트를 위한 UI/UX 디자인 가이드입니다.

## 왜 디자인 시스템이 필요한가?

디자인 시스템은 **일관된 사용자 경험**을 만들기 위한 규칙과 컴포넌트의 모음입니다.

### 디자인 시스템의 장점

| 장점 | 설명 |
|------|------|
| **일관성** | 모든 페이지에서 동일한 룩앤필 제공 |
| **효율성** | 재사용 가능한 컴포넌트로 개발 속도 향상 |
| **유지보수** | 한 곳에서 수정하면 전체에 반영 |
| **협업** | 디자이너-개발자 간 명확한 기준 공유 |

## 핵심 원칙

### 1. 일관성 (Consistency)

> 같은 요소는 항상 같은 모습과 동작을 가져야 합니다.

- 버튼은 항상 같은 스타일
- 같은 종류의 텍스트는 같은 크기와 색상
- 동일한 간격 규칙 적용

### 2. 계층 구조 (Hierarchy)

> 중요한 것이 먼저 눈에 들어와야 합니다.

- 제목 > 부제목 > 본문 순서로 크기 차이
- 주요 버튼은 눈에 띄는 색상
- 여백으로 콘텐츠 그룹 구분

### 3. 단순함 (Simplicity)

> 불필요한 요소는 제거합니다.

- 색상은 3-5가지만 사용
- 폰트는 2-3개만 사용
- 여백을 충분히 활용

### 4. 접근성 (Accessibility)

> 모든 사용자가 사용할 수 있어야 합니다.

- 충분한 색상 대비
- 읽기 쉬운 폰트 크기
- 키보드로도 조작 가능

## 빠른 참조

### 색상 (60-30-10 규칙)

```
60% - 배경색 (흰색, 회색 등 중립적인 색)
30% - 보조색 (카드, 섹션 배경)
10% - 강조색 (버튼, 링크, CTA)
```

### 폰트 크기

```
제목 (h1):     32-48px
부제목 (h2):   24-32px
소제목 (h3):   18-24px
본문:          16-18px
작은 텍스트:   12-14px
```

### 간격 (8px 단위)

```
xs:  4px   (0.25rem)
sm:  8px   (0.5rem)
md:  16px  (1rem)
lg:  24px  (1.5rem)
xl:  32px  (2rem)
2xl: 48px  (3rem)
```
