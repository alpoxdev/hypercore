# Web Pack

병목이 프론트엔드 렌더링, 번들 크기, 내비게이션 속도, 사용자 가시 웹 성능에 있을 때 이 팩을 사용한다.

## 권장 프롬프트

1. `메인 라우트의 로딩 비용을 줄이고 눈에 보이는 동작은 그대로 유지해줘.`
2. `수정 전에 번들 크기, Core Web Vitals 프록시, 회귀 안전성을 벤치마크해줘.`
3. `이 프론트엔드 병목을 측정 가능한 반복 실험으로 개선하고 점수 오르는 변경만 남겨줘.`

## 권장 Proof Command

- production build
- bundle 분석 명령
- smoke route 또는 E2E 점검
- 기존 perf script 또는 synthetic vitals 점검

## 권장 이진 Eval

- 선택한 proof set이 웹 병목에 적절한가?
- 실행이 대상 라우트의 눈에 보이는 동작을 유지하는가?
- 번들 크기, LCP 프록시, INP 프록시, request 수 같은 구체적 자원 지표를 추적하는가?
- 아티팩트가 소유 라우트 또는 앱 범위를 분명히 기록하는가?
- 반복된 무회귀 점검 없이 성공을 주장하지 않는가?
