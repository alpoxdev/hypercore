# 점진적 공개

**목적**: 스킬을 얇게 유지하면서도 깊은 상세 정보를 필요할 때 찾을 수 있게 합니다.

## 1. 3단계 구조

다음 로딩 모델을 사용합니다.

1. metadata
2. 코어 `SKILL.md`
3. rules, references, scripts, assets

아래 계층으로 갈수록 내용은 더 구체적이고 더 선택적이어야 합니다.

## 2. 코어는 얇게 유지

코어 스킬에는 다음을 둡니다.

- 스킬의 일
- 트리거
- 워크플로
- 더 깊은 파일로 가는 포인터

다음은 밖으로 뺍니다.

- 긴 예시
- 공식 출처 요약
- 도메인 스키마
- 변형별 상세 내용

## 3. 한 단계 깊이 선호

가능하면 `SKILL.md`에서 직접 가리키는 references를 사용합니다.
깊게 중첩된 참조 체인은 피합니다.

## 4. 필요에 따라 분리

다음 기준으로 내용을 나눕니다.

- 반복 정책은 `rules/`
- 상세 지식은 `references/`
- 결정적 실행은 `scripts/`
- 출력 자원은 `assets/`

## 5. 재독 점검

분리 후에는:

- 코어만 읽어도 스킬이 이해되는지
- 보조 파일을 쉽게 찾을 수 있는지
- 핵심 지시가 여러 계층에 중복되지 않았는지

를 확인합니다.
