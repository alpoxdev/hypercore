# 스킬 안티패턴

**목적**: 스킬 작성에서 자주 실패하는 패턴을 막습니다.

## 피할 것

- 너무 모호해서 안정적으로 트리거되지 않는 설명 문구
- 미니 위키처럼 비대해진 `SKILL.md`
- 코어와 references 사이의 중복 상세 내용
- 찾기 어렵게 깊게 중첩된 references
- 스킬 안에 불필요하게 만드는 `README.md`, `CHANGELOG.md`, `QUICK_REFERENCE.md`
- 코어 본문에 넣은 시간 민감한 공급자 세부사항
- 신뢰성 근거 없이 추가한 scripts
- 스킬이 경로를 추천해야 하는데 선택지만 너무 많이 늘어놓는 경우

## 위험 신호

- "이 스킬은 많은 일을 돕는다"
- 가이드 없이 "다섯 가지 접근법이 있다"만 적는 경우
- 여러 파일에 같은 정의가 반복됨
- 현재 코어 규칙에 오래된 공급자 가이드가 섞여 있음
- "구조가 좋아 보인다"는 이유로 검증을 생략함
