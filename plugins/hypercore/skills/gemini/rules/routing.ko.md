# 라우팅 규칙

사용자가 Gemini CLI 자체, Gemini 세션, 또는 협업 모델로서의 Gemini를 명시적으로 원할 때만 Gemini를 사용한다.

요청이 실제로 Gemini CLI를 필요로 하지 않으면 다른 경로로 전환한다.

## 범위 안

- 사용자가 Gemini를 요청했고, 리서치, 추론, AI 지원을 위해 Gemini CLI를 실행해야 할 때
- `--resume`, `--list-sessions` 로 Gemini 세션을 재개하거나 확인해야 할 때
- 사용자가 Gemini가 파일을 수정하길 명시적으로 원할 때만 편집 작업을 맡길 때

## 범위 밖

- 사용자가 Gemini를 요청하지 않은 일반 문서 작성이나 런북 정리
- Gemini를 쓰지 않아도 되는 직접 로컬 편집
- Gemini CLI 자체가 필요하지 않은 스킬 생성 또는 스킬 리팩터링

요청이 실제로 Gemini CLI를 필요로 하지 않으면 직접 편집이나 다른 스킬로 넘긴다.

일반 문서 작성, 런북 수정, 무관한 로컬 편집에는 Gemini CLI 명령을 만들지 않는다.
Gemini가 잘못된 도구라면 `auto_edit` 나 `--yolo` 로 승인 강도를 올리지 않는다.
Gemini가 맞지 않는 작업이면 억지로 실행하지 말고 깔끔하게 전환한다.
