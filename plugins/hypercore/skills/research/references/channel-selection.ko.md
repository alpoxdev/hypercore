# Channel Selection

코어 스킬이 증거 채널을 고르라고 할 때 이 참고 파일을 사용합니다.

## 주제별 채널 매핑

| 주제 형태 | 시작 채널 | 필요 시 추가 | 피할 것 |
|------|------|------|------|
| 내부 프로젝트 동작 또는 이력 | 로컬 저장소 검색 | 저장소 메타데이터가 중요하면 git 이력 또는 GitHub 근거 추가 | 처음부터 라이브 웹 검색 |
| 라이브러리 또는 패키지 사용법 | 공식 문서 또는 Context7 (Claude Code) / WebFetch (Codex/기타) | GitHub 이슈, 릴리스, 예제 | 문서보다 일반 블로그 우선 |
| 제품, 저장소, 릴리스 비교 | GitHub 릴리스, 이슈, 변경로그, 코드 검색 | 독립적 해설이 필요하면 라이브 웹 | 마케팅 페이지만 믿기 |
| 시장, 트렌드, 시사성 조사 | 날짜 인식이 있는 라이브 웹 검색 | 공식 보고서나 기업 공시 | 오래된 에버그린 글 |
| 학술 또는 개념 조사 | 원문 논문, 공식 레퍼런스 | 종합을 위한 2차 설명 자료 | 의견글을 1차 근거처럼 쓰기 |
| 다중 옵션 기술 비교 | 각 옵션의 공식 문서, 그다음 독립 벤치마크/사례 | 구현 현실 확인용 GitHub 근거 | 기준표 없는 결론 |

## 선택 메모

- 질문을 충분히 답할 수 있는 가장 작은 채널 세트를 우선합니다.
- 사용자가 current, latest, 최신 같은 표현을 쓰면 반드시 라이브 검색을 포함합니다.
- 코드베이스 전용 질문은 외부 근거가 꼭 필요할 때만 저장소 밖으로 나갑니다.
- 패키지 버전이 명시되면 해당 버전의 공급자 또는 공식 문서에서 시작합니다.
