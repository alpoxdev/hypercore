# Execution Rules

## Required inputs

아이데이션 전에 다음을 수집한다:

- 문제 또는 기회 진술
- 대상 사용자 또는 시장
- 의도한 결과 또는 결정
- 시간, 팀, 예산, 규제 같은 엄격한 제약

핵심 제약이 빠져 있다면 명시적 가정으로만 진행한다.

## Operating sequence

1. 문제를 다시 진술하고 그에 맞는 2-3개 프레임워크를 고른다.
2. 아이디어를 생성하기 전에 문제를 재정의한다.
3. 적어도 3개의 뚜렷한 방향에 걸쳐 다양한 선택지를 생성한다.
4. 중복과 얇은 변형을 제거한다.
5. 명시적 근거, 가정, 다음 테스트로 우선순위를 정한다.

## Anti-patterns

- 필수 글자 수를 채우려고 답변을 부풀리지 않는다.
- 몇 개만 관련 있을 때 모든 프레임워크를 쏟아내지 않는다.
- 트레이드오프를 설명하지 않고 아이디어에 점수를 매기지 않는다.
- 수량만으로 창의성을 판단하지 않는다.
