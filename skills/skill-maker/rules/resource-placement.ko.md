# 자원 배치

**목적**: 새 스킬 내용을 추가하기 전에 어디에 둬야 하는지 결정합니다.

## 판단 순서

다음 질문을 순서대로 봅니다.

1. 이것이 스킬의 핵심 정체성이나 트리거인가?
2. 이것이 재사용 가능한 정책인가?
3. 필요할 때만 불러오는 상세 지식인가?
4. 설명보다 결정적 실행이 더 나은가?
5. reasoning 컨텍스트보다 출력 자원에 가까운가?

## 배치 규칙

- 스킬의 일, 트리거, 상위 워크플로를 정의하면 `SKILL.md`에 둡니다.
- 재사용 정책, 검증, 반복 판단 가이드면 `rules/`에 둡니다.
- 상세 지식, 공식 문서, 스키마, 변형별 노트면 `references/`에 둡니다.
- 결정적 실행이 설명보다 더 신뢰할 수 있으면 `scripts/`에 둡니다.
- 결과물 생성을 지원하는 파일이면 `assets/`에 둡니다.

## 빠른 점검

- 이것을 빼면 스킬의 일이 अस्पष्ट해지면 `SKILL.md`일 가능성이 큽니다.
- 반복해서 판단하는 방법을 설명하면 `rules/`일 가능성이 큽니다.
- 크고 선택적이면 `references/`일 가능성이 큽니다.
- 절차적이고 깨지기 쉬우면 `scripts/`일 수 있습니다.
- 복사, 채우기, 삽입되는 출력 자원이면 `assets/`일 가능성이 큽니다.
