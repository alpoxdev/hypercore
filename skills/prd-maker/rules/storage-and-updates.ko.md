# 저장과 갱신

스킬이 파일을 만들거나 바꿀 때 이 규칙 파일을 사용합니다.

## 1. 폴더 위치

각 PRD는 여기 저장합니다.

`.hypercore/prd/[slug]/`

기본 파일:

- `prd.md`
- `sources.md`

## 2. Slug 규칙

- 짧은 ASCII kebab-case slug를 우선합니다.
- 전체 요청 문장 말고 이니셔티브나 기능 이름을 기준으로 만듭니다.
- 같은 주제를 갱신하는 요청이면 기존 slug를 재사용합니다.

좋은 예:

- `billing-retry-flow`
- `team-inbox-assignment`

피할 것:

- 문장 전체를 slug로 쓰는 것
- 날짜가 제품 개념의 일부가 아닌데 날짜를 slug에 넣는 것
- 같은 이니셔티브에 폴더를 여러 개 만드는 것

## 3. 파일 역할

- `prd.md`: 살아있는 요구사항 문서
- `sources.md`: 근거 로그, 쿼리 로그, 소스 메모

사용자가 명시적으로 요구하지 않는 한 README, notes, changelog 같은 추가 파일은 만들지 않습니다.

## 4. 생성 흐름

폴더가 없을 때는:

- 폴더를 만듭니다
- 템플릿 자산으로 `prd.md`를 만듭니다
- 템플릿 자산으로 `sources.md`를 만듭니다
- 모르는 내용은 조용히 비우지 말고 placeholder 또는 오픈 질문으로 남깁니다

## 5. 갱신 흐름

폴더가 이미 있으면:

- 현재 `prd.md`와 `sources.md`를 먼저 읽습니다
- 바뀐 섹션만 수정합니다
- 기존 링크, 기존 결정, 유용한 맥락은 보존합니다
- `prd.md`의 변경 이력 섹션에 날짜가 있는 항목을 추가합니다
- `sources.md`에는 기존 근거를 지우기보다 새 근거를 덧붙입니다. 명확히 폐기된 경우에만 교체합니다

## 6. 폴더가 없을 때의 복구

사용자가 PRD 갱신을 요구했는데 맞는 폴더가 없으면:

- 새 폴더를 만듭니다
- 기준 문서가 없었다는 점을 PRD 변경 이력에 가정으로 남깁니다
- `create` 모드로 계속 진행합니다
