# 검증

PRD 작업이 끝났다고 말하기 전에 이 체크리스트를 사용합니다.

## 필수 체크

- 요청이 일반 계획, 일반 문서, 조사 메모가 아니라 실제 PRD를 요구했다.
- 폴더가 `.hypercore/prd/[slug]/` 아래에 존재한다.
- `prd.md`와 `sources.md`가 모두 존재한다.
- PRD에 목표, 범위, 비목표, 요구사항, 메트릭, 리스크/의존성, 오픈 질문, 변경 이력이 명시되어 있다.
- 조사 근거가 들어간 비자명한 주장에는 링크가 있다.
- `sources.md`에 서로 다른 쿼리 기록이 있거나, 외부 조사가 불필요했다는 명시적 기록이 있다.
- PRD를 갱신한 경우 변경 이력에 무엇이 왜 바뀌었는지 드러난다.
- 미지수는 조용히 빠지지 않고 오픈 질문 또는 가정으로 남아 있다.

## 품질 체크

- PRD가 빠르게 훑어볼 수 있을 만큼 압축되어 있다.
- 메인 문서는 의사결정 중심이고, 근거 누적은 evidence log에 있다.
- 범위가 충분히 명확해서 엔지니어링과 디자인이 포함/제외를 구분할 수 있다.
- 요구사항은 정말 필요한 제약이 아닌 한 구현 지시가 아니라 제품 동작 또는 결과 중심으로 적혀 있다.

## 전방 테스트 질문

- 새 팀원이 이 이니셔티브가 어떤 문제를 푸는지 이해할 수 있는가?
- 엔지니어링이 필수 요구사항과 의도적 비범위를 구분할 수 있는가?
- 한 달 뒤 다시 봤을 때 무엇이 바뀌었는지 알 수 있는가?
- 근거가 바뀌면 문서 전체를 갈아엎지 않고 근처 source log만 새로 고칠 수 있는가?
