# Report Template

Use this reference when you are drafting the saved research report.

## File Path

Save to:

`.hypercore/research/[NN].slug.md`

Example:

`.hypercore/research/03.websocket-vs-sse.md`

## Template

```markdown

> Date reviewed: YYYY-MM-DD | Depth: quick/standard/deep | Sources reviewed: N | Sources cited: M
```

## Writing Rules

- Put the answer before the backstory.
- Link every non-obvious claim.
- Use exact dates when recency matters.
- Keep caveats explicit instead of hiding them in prose.
- For standard, deep, or parallel research, include the source ledger and claim-source matrix.
- Count duplicate sources once, even if multiple lanes found them.
