# Framework Reference

## First-principles core

Use this split:

- `A`: real constraints such as physics, hard technical limits, legal limits, or hard dependencies
- `B`: conventions, defaults, habits, or copied industry patterns
- `C`: uncertain assumptions that need evidence

## Socratic question set

Apply some of these to the main assumptions:

- What exactly does this claim mean?
- What are we assuming?
- What evidence proves it?
- What other framing is possible?
- If this assumption is false, what changes?
- Why did we accept this as normal?

## Rebuild sequence

1. Keep only the A constraints.
2. Remove or challenge the B conventions.
3. Convert C items into evidence questions.
4. Design alternatives using only surviving truths.

## Stress-test methods

- `Inversion`: how would we guarantee failure?
- `Pre-mortem`: if this fails in six months, what most likely caused it?
