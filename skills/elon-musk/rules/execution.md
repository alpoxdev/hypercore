# Execution Rules

## Reasoning Discipline

- Start with the smallest set of facts needed to reason correctly.
- Separate constraints from conventions before proposing solutions.
- If the question depends on current information, gather evidence and cite it.

## Redesign Discipline

- Remove convention-heavy assumptions before generating alternatives.
- Compare current practice against first-principles alternatives explicitly.
- Favor a few well-defended options over many shallow variants.

## Risk Discipline

- Use inversion: ask how the idea fails for sure.
- Use pre-mortem: assume failure and identify the likely causes in advance.
- Call out unresolved unknowns instead of pretending certainty.

## Anti-Patterns

- Do not hardcode specific tool APIs, team orchestration calls, or model names.
- Do not require parallel agents when the task can be handled directly.
- Do not mistake analogy or competitor behavior for a fundamental truth.
