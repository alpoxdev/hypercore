# 라우팅 규칙

사용자가 Codex CLI 실행, Codex 세션 재개, 또는 코딩 협업자로서의 Codex를 명시적으로 원할 때만 이 스킬을 사용한다.

요청이 실제로 Codex CLI를 필요로 하지 않으면 범위를 벗어난다.

## 범위 안

- 분석, 리뷰, 리팩토링 지원을 위해 `codex exec`를 실행한다.
- `codex exec ... resume --last`로 최근 Codex 세션을 이어간다.
- Codex CLI 자체가 요청된 워크플로우의 일부일 때 코드 점검이나 패치를 맡긴다.

## 범위 밖

- 스킬 생성 또는 스킬 리팩터링
- 런북이나 일반 문서 작성
- 사용자가 Codex를 요청하지 않았고 직접 편집이 더 적절한 작업

Codex CLI 자체가 필요하지 않으면 다른 스킬이나 직접 편집으로 전환한다.

일반 문서 작성, 런북 정리, 스킬 생성에는 Codex 커맨드를 만들지 않는다.
Codex가 맡을 일이 아닌데도 샌드박스를 올려서까지 억지로 Codex를 끼워 넣지 않는다.
Codex가 맞지 않는 작업이면 실행하지 말고 깔끔하게 전환한다.
