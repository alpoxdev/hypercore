# Monorepo Pack

병목이 워크스페이스 전체 빌드 비용, 중복 설정, 패키지 경계 마찰, 다중 패키지 정리에 있을 때 이 팩을 사용한다.

## 권장 프롬프트

1. `이 모노레포 병목을 측정 가능한 반복 실험으로 개선하고 점수 오르는 변경만 남겨줘.`
2. `수정 전에 워크스페이스 빌드 비용, task fan-out, 회귀 안전성을 벤치마크해줘.`
3. `패키지 경계를 유지하면서 cross-package drag를 줄여줘.`

## 권장 Proof Command

- workspace build
- 영향받는 패키지 테스트
- package graph 또는 task runner 출력
- 소유 패키지 그룹용 smoke 명령

## 권장 이진 Eval

- 선택한 proof set이 모노레포 또는 워크스페이스 병목에 적절한가?
- 실행이 패키지 경계를 유지하고 cross-repo 혼합을 피하는가?
- affected task 수, build 시간, 제거된 중복 설정 같은 구체적 워크스페이스 지표를 추적하는가?
- 아티팩트가 소유 워크스페이스 범위를 분명히 기록하는가?
- 반복된 무회귀 점검 없이 성공을 주장하지 않는가?
