# AGENTS.md

이 파일은 Claude Code, Codex, Cursor 등 모든 AI 에이전트 CLI에서 공통으로 읽는 프로젝트 규칙입니다.
범위는 현재 프로젝트 루트와 그 하위 디렉터리 전체입니다.

## 작업 범위

- 항상 현재 프로젝트 범위 안에서만 작업한다.
- 조사, 수정, 생성, 참조 대상은 이 저장소 내부 파일로 제한한다.
- 홈 디렉터리의 전역 에이전트 설정이나 전역 스킬 디렉터리를 작업 근거로 사용하지 않는다.
- 특히 홈 디렉터리 아래의 전역 에이전트/스킬 디렉터리(예: `~/.agents/`, 각 CLI 도구의 전역 설정)는 읽거나 수정하거나 참조하지 않는다.
- 이 프로젝트 내부에 존재하는 `skills/`, `instructions/`, `scripts/`, `README.md` 같은 로컬 파일만 우선 사용한다.

## `npx skills add` 관련 규칙

- `skills` 공식 문서 기준으로 `npx skills add`의 기본 설치 범위는 전역이 아니라 프로젝트다.
- 전역 설치는 `-g` 또는 `--global`을 명시했을 때만 해당한다.
- 따라서 이 프로젝트에서 작업할 때는 전역 스킬 설치 위치를 기준으로 판단하지 않는다.
- `npx skills add`로 설치된 스킬을 고려해야 할 경우에도 현재 프로젝트 내부 경로만 본다.
- 예를 들어 프로젝트 내부의 `.agents/skills/`, `.claude/skills/`, `.codex/skills/`는 로컬 범위이므로 허용되지만, `~/` 아래 동일 이름 경로는 범위 밖이다.

## 우선순위

- 프로젝트 로컬 규칙이 전역 개인 설정보다 우선한다.
- 필요한 정보가 저장소 안에 없으면, 먼저 저장소 내부 문서와 코드에서 해결하고, 그래도 없을 때만 사용자에게 확인한다.

## 변경 원칙

- 변경은 작고 되돌리기 쉽게 유지한다.
- 현재 요청과 무관한 전역 환경 정리는 하지 않는다.
- 전역 스킬/설정 상태를 가정하지 않는다.

## 플랫폼별 참고

- 이 저장소의 스킬은 Claude Code, Codex, Cursor에서 사용할 수 있다.
- 일부 스킬은 특정 CLI 전용이다 (예: `claude-code` → Claude Code 전용, `codex` → Codex 전용, `gemini` → Gemini 전용).
- 각 스킬의 `SKILL.md` 내 `compatibility` 필드에서 플랫폼 호환성을 확인할 수 있다.
