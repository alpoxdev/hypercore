# 반응형 디자인 가이드 (2025-2026)

> 브레이크포인트, Container Queries, Fluid Typography 디자인 사양서

---

## 1. 브레이크포인트

### 표준 브레이크포인트

| 이름 | 너비 | 대상 기기 |
|------|------|----------|
| **xs** | 0-359px | 소형 폰 |
| **sm** | 360-639px | 일반 폰 |
| **md** | 640-767px | 대형 폰/소형 태블릿 |
| **lg** | 768-1023px | 태블릿 |
| **xl** | 1024-1279px | 소형 데스크톱/랩탑 |
| **2xl** | 1280-1535px | 데스크톱 |
| **3xl** | 1536px+ | 대형 데스크톱 |

### 디자인 권장 브레이크포인트

| 접근법 | 브레이크포인트 | 용도 |
|--------|--------------|------|
| **Mobile-First** | 360 → 768 → 1024 → 1440 | 일반적 |
| **Desktop-First** | 1440 → 1024 → 768 → 360 | 레거시 |
| **Content-First** | 콘텐츠 깨지는 지점 | 최적 |

## 2. Container Queries

### 컨테이너 사이즈

| 이름 | 최소 너비 | 용도 |
|------|----------|------|
| **@xs** | 160px | 작은 위젯 |
| **@sm** | 320px | 카드 내부 |
| **@md** | 448px | 중간 컨테이너 |
| **@lg** | 512px | 넓은 컨테이너 |
| **@xl** | 576px | 전체 섹션 |
| **@2xl** | 672px | 메인 콘텐츠 |

### Container Query 단위

| 단위 | 설명 | 사용 예시 |
|------|------|----------|
| **cqw** | 컨테이너 너비 1% | 폰트 크기 유동화 |
| **cqh** | 컨테이너 높이 1% | 높이 기반 레이아웃 |
| **cqi** | 인라인 축 1% | 반응형 패딩 |
| **cqb** | 블록 축 1% | 반응형 마진 |
| **cqmin** | min(cqi, cqb) | 최소값 기반 |
| **cqmax** | max(cqi, cqb) | 최대값 기반 |

## 3. Fluid Typography

### 폰트 크기 스케일

| 역할 | 모바일 | 데스크톱 | Fluid 범위 |
|------|--------|----------|-----------|
| **Display** | 36px | 72px | 36-72px |
| **H1** | 30px | 48px | 30-48px |
| **H2** | 24px | 36px | 24-36px |
| **H3** | 20px | 24px | 20-24px |
| **Body** | 16px | 18px | 16-18px |
| **Small** | 14px | 14px | 고정 |
| **Caption** | 12px | 12px | 고정 |

### Fluid 계산 공식

| 뷰포트 범위 | 공식 |
|------------|------|
| **360-1440px** | `clamp(최소, 계산값, 최대)` |
| **선호값 계산** | `최소 + (최대-최소) × ((100vw - 360px) / (1440-360))` |

### Line Height 권장값

| 폰트 크기 | Line Height |
|----------|-------------|
| **12-14px** | 1.5-1.6 |
| **16-18px** | 1.5-1.7 |
| **20-24px** | 1.4-1.5 |
| **30px+** | 1.2-1.3 |

## 4. 레이아웃 패턴

### 그리드 시스템

| 브레이크포인트 | 열 수 | 거터 | 마진 |
|---------------|------|------|------|
| **Mobile** | 4 | 16px | 16px |
| **Tablet** | 8 | 24px | 32px |
| **Desktop** | 12 | 24px | auto (max 1200px) |

### 컴포넌트별 열 점유

| 컴포넌트 | 모바일 | 태블릿 | 데스크톱 |
|---------|--------|--------|----------|
| **카드** | 4열 (100%) | 4열 (50%) | 3-4열 (25-33%) |
| **사이드바** | 숨김 | 2열 | 3열 (고정) |
| **콘텐츠** | 4열 | 6열 | 9열 |

### 스택 vs 나란히

| 상황 | 모바일 | 태블릿+ |
|------|--------|--------|
| **이미지+텍스트** | 세로 스택 | 나란히 (50:50) |
| **폼 필드** | 전체 너비 | 2열 또는 3열 |
| **카드 그리드** | 1열 | 2-4열 |
| **네비게이션** | 햄버거 메뉴 | 가로 메뉴 |

## 7. 터치 vs 마우스

### 인터랙션 차이

| 요소 | 터치 (모바일) | 마우스 (데스크톱) |
|------|-------------|------------------|
| **터치 타겟** | 44-48px 최소 | 24px 가능 |
| **호버 상태** | 없음 | 필수 |
| **드래그** | 스와이프 제스처 | 드래그 앤 드롭 |
| **툴팁** | 탭 앤 홀드 | 호버 |

### 네비게이션 변환

| 데스크톱 | 모바일 변환 |
|---------|------------|
| **가로 메뉴** | 햄버거 메뉴 |
| **드롭다운** | 전체 화면 드로어 |
| **사이드바** | 바텀 시트 또는 숨김 |
| **호버 메뉴** | 탭 메뉴 |

## 8. 2025-2026 트렌드

### 새로운 디바이스 대응

| 디바이스 | 고려사항 |
|---------|---------|
| **Foldable** | 힌지 영역 피하기, 연속성 유지 |
| **Ultra-wide** | 최대 너비 제한, 다중 열 |
| **AR/VR** | 3D 공간 UI, 시선 기반 인터랙션 |

### 새로운 CSS 기능

| 기능 | 용도 |
|------|------|
| **Container Queries** | 컴포넌트 기반 반응형 |
| **:has()** | 부모 선택자 |
| **Subgrid** | 중첩 그리드 정렬 |
| **CSS Nesting** | 선택자 중첩 |
| **@layer** | 캐스케이드 레이어 |

### Intrinsic Design 원칙

| 원칙 | 설명 |
|------|------|
| **콘텐츠 기반** | 브레이크포인트가 아닌 콘텐츠에 따라 |
| **유동적 스케일** | 단계별이 아닌 연속적 변화 |
| **컴포넌트 독립** | 컨테이너 기반 자체 반응형 |
